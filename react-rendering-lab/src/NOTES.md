# React Rendering Lab

## 챕터 1. 렌더링은 언제 일어나는가?

### 렌더링이란?

React 컴포넌트 함수가 실행되는 것.
함수가 실행된다 = 렌더링이 일어난다.
실제 DOM이 바뀌는 것과는 다름!

### 렌더링 트리거 4가지

1. 컴포넌트 최초 마운트
2. state 변경 (useState의 setter)
3. 부모 컴포넌트 리렌더링 → 자식도 리렌더링
4. Context 값 변경

### 핵심 원칙: 참조 동일성

React는 이전 값과 새 값을 Object.is() 로 비교한다.
값이 같으면 렌더링 안 함. 값이 다르면 렌더링 함.

### 오늘 실험

#### 실험 1. +1 증가 버튼

- count가 0 → 1로 바뀜
- 값이 달라짐 → 렌더링 O
- 콘솔에 🔴 App 렌더링 됨! 찍힘

#### 실험 2. 그대로 버튼

- count가 0인데 0으로 세팅
- 값이 똑같음 → 렌더링 X
- 콘솔에 아무것도 안 찍힘

### 한 줄 정리

> 값이 바뀌어야 렌더링이 일어난다. 같은 값이면 React가 스킵한다.
